# 04 · Communication & creative brief

At S level: you own content + design. No comm planning or media planning needed.
At M level: adds communication planning and media planning as formal deliverables.

The prompts below are split accordingly.

---

## S LEVEL

### Prompt 09-S · Plan communication for an S campaign

**When to use:** S campaign is defined. Need to plan in-app and owned channel communication — no paid media.

```
I need a communication plan for an S-level campaign.

Campaign level: S
Available channels: in-app (banner, push notification) + owned out-app
                   (owned social, CRM — no paid media)
Asset scope: content copy + design (not concept creative)
Prep timeline: [3–4 weeks for simple / 5–6 weeks for complex assets]

Campaign:
- Theme: [brief description]
- Core mechanic: [what the user does and gets]
- Audience: [segment]
- Duration: [start – end date]
- Launch date: [target]

Please plan:
1. Key milestones from today to launch — within prep timeline
2. Message hierarchy: one primary message, one supporting message
3. In-app push schedule: frequency, time of day, content per wave
   (don't over-notify — recommend max frequency for this audience)
4. In-app banner: placement, message, timing
5. Owned out-app: which owned channel, what content, when
6. One thing to A/B test if time allows
```

---

### Prompt 10-S · Write push notification copy by segment

**When to use:** Need variants fast for an S campaign. No paid amplification — the notification itself must carry the full weight.

```
Campaign level: S (push notification is primary channel — no paid backup)

Segment: [behavior, balance tier, recency]
Promotion: [what the user gets and conditions]
Tone: [casual / warm / direct]

Technical limits:
- Title: max 40 characters
- Body: max 80 characters

Write 5 variants — different angles:
urgency / benefit / curiosity / personalization / social proof

For each:
- Character count
- Angle being used
- Why it fits this segment

Which 2 to test first and why?

Note: at S level there's no paid retargeting fallback —
if push doesn't convert, there's no second touch.
Factor this into which angle to prioritize.
```

---

### Prompt 11-S · Write a design brief for an S campaign

**When to use:** Briefing in-house creative team directly (no external agency at S level).

```
Campaign level: S
Asset type: [in-app banner / push thumbnail / owned social post]
Working with: in-house creative team
Timeline: [weeks until launch]

Campaign: [name / theme]
Context: [tone, audience, season or trigger]

My rough idea: [free-form — colors, imagery, emotion, reference]
What I don't want: [specific things to avoid]

Write a complete brief:
1. Objective — what this visual needs to make the user do
2. Audience insight — who sees this, where in the app, what they're doing
3. Key message — one sentence only
4. Visual direction — specific enough for in-house team to start without questions
5. Exact copy on the visual — don't leave it open
6. Specs — dimensions, format, deadline
```

## M LEVEL

### Prompt 09-M · Plan communication for an M campaign

**When to use:** M campaign confirmed. Needs formal comm planning and media planning as deliverables.

```
Campaign level: M
Available channels: in-app + owned out-app + [1–2 paid media channels — specify]
Asset scope: creative content + design + communication planning + media planning
Prep timeline: approximately 2 months

Campaign:
- Theme: [description]
- Core mechanic: [what the user does and gets]
- Audience: [primary segment + secondary if any]
- Duration: [start – end]
- Budget: [approximate — confirms M level < 1B/month]

Please produce:
1. Communication timeline — phases with milestones:
   pre-launch / launch / mid-campaign / closing / post
2. Message architecture:
   - Primary message (campaign level)
   - Channel-specific adaptations (in-app vs. paid vs. owned social)
3. Media planning input:
   - Which 1–2 paid channels to recommend and why
   - Suggested budget split between paid and owned
   - KPI to monitor per channel (media KPI, not just business KPI)
4. Push + in-app plan: same as S level but coordinated with paid timeline
5. What to A/B test and in which phase
```

---

### Prompt 12 · Review a design through a growth lens (any level)

**When to use:** Design received from team. Need structured feedback — not aesthetic opinion, but conversion and communication review.

```
Campaign level: [S / M]
Asset type: [banner / push thumbnail / social post]

Goal of this visual: [what the user should do after seeing it]
Placement: [where exactly in the app or channel]
Audience mindset: [what they're doing when they see this]

Review across 4 criteria:
1. Clarity — does the user immediately know what to do?
2. Hierarchy — where does the eye go first? Is the key message prominent?
3. CTA — clear and compelling enough for this audience?
4. Context fit — right for the placement and mindset?

For each: good / needs work + specific suggestion if improvement needed.
No aesthetic feedback unless it directly affects conversion.
```
