# 03 · Game Mechanics & Engagement / Cơ chế game & engagement

**EN:** Engagement loops bring users back without relying on vouchers alone. These prompts help you design game mechanics at the right complexity for your campaign level, and diagnose where users drop off in an existing flow.

**VI:** Engagement loop giúp user quay lại mà không cần phụ thuộc hoàn toàn vào voucher. Các prompt này giúp bạn thiết kế cơ chế game đúng với level campaign, và tìm ra user đang dừng lại ở bước nào trong flow hiện tại.

## Key principle / Nguyên tắc chính

**EN:** Game mechanics at S level must be self-contained — they work without paid media amplification. At M level, a paid channel can drive discovery, but the loop itself still needs to be compelling enough to sustain without it.

**VI:** Cơ chế game ở level S phải tự vận hành được — không cần paid media khuếch đại. Ở level M, paid channel giúp user tìm thấy game, nhưng bản thân loop phải đủ hấp dẫn để giữ user mà không cần tiếp tục chi paid.

## Example output / Ví dụ output

**Prompt 07 — S level, audience: mainstream users unfamiliar with loyalty**

```
Mechanic A (Simple — recommended for S):
Daily check-in streak
- User opens app → taps "check in" → earns small points
- Day 7 streak = bonus reward
- Return loop: streak anxiety (don't want to break it)
- Point cost: low (5–10 points/day)
- Asset needed: in-app banner + 1 animation = fits S level

Mechanic B (Medium — better for M):
Mini quiz about deals
- Weekly quiz about current promotions → earn points for correct answers
- Drives awareness of voucher catalog while rewarding engagement
- Needs comm planning to announce each week → M level

Mechanic C (Complex — L level only):
Collaborative challenge
- Team up with friends, reach a collective goal
- High engagement but needs concept creative + significant dev → not S or M

Recommendation: Mechanic A for S campaign
Risk: streak breaks feel punishing — add "streak freeze" option
MEU metric: % of MAU who check in at least 3x in 30 days
```
