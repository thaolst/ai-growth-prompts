## 📖 Glossary — Danh sách thuật ngữ

### Chiến lược & Framework

| Thuật ngữ | Định nghĩa ngắn | Ví dụ |
|---|---|---|
| **AARRR (Pirate Metrics)** | 5 giai đoạn user journey: Acquisition → Activation → Retention → Revenue → Referral | Dùng để chẩn đoán đâu là bottleneck trong funnel |
| **North Star Metric** | 1 metric duy nhất đo giá trị user nhận được nếu dùng sản phẩm đúng cách | Duolingo: "Số phút học mỗi ngày" — không phải "số user đăng ký" |
| **Growth Loop** | Hệ thống tự tái tạo: output của 1 cycle là input cho cycle tiếp theo | Dropbox: Dùng → Share → Nhận thêm dung lượng → Dùng tiếp |
| **JTBD (Jobs to Be Done)** | User "thuê" sản phẩm để làm 1 công việc cụ thể trong đời | Người thuê Grab không phải để "đi xe" mà để "đi từ A đến B không phải lo gì" |
| **ICE/RICE** | Khung ưu tiên thí nghiệm: Impact × Confidence × Ease (ICE) hoặc Reach × Impact × Confidence × Effort (RICE) | Campaign có Impact 8, Confidence 6, Ease 7 → ICE = 8×6×7 = 336 |
| **Growth Funnel (Phễu)** | Các bước user đi từ biết đến sản phẩm → chuyển đổi | Install → Register → First action → Retention → Purchase |
| **Flywheel** | Mô hình vòng quay: khách hàng hài lòng → referral → khách hàng mới → hài lòng hơn | HubSpot: dùng content marketing (khách hàng đọc → tin tưởng → mua → recommend) |
| **TAM/SAM/SOM** | TAM = Tổng thị trường, SAM = Thị trường có thể phục vụ, SOM = Thị trường thực tế chiếm được | Dùng để estimate campaign scale và justify budget |

### Metric & Đo lường

| Thuật ngữ | Định nghĩa ngắn | Ví dụ |
|---|---|---|
| **CAC (Customer Acquisition Cost)** | Tổng chi phí marketing / Số user mới | Nếu campaign tốn 100 triệu, được 10k user → CAC = 10k |
| **LTV (Lifetime Value)** | Tổng doanh thu trung bình 1 user mang về trong toàn bộ vòng đời | User app game: nạp 50k/tháng × 12 tháng = LTV = 600k |
| **Retention Rate** | % user vẫn dùng sản phẩm sau X ngày | D7 retention = 30% → 30% user quay lại trong 7 ngày |
| **Churn Rate** | % user rời bỏ sản phẩm trong 1 khoảng thời gian | Hàng tháng mất 5% user → annual churn = 1 - 0.95^12 ≈ 46% |
| **DAU/MAU (Daily/Monthly Active Users)** | Số user hoạt động trong ngày / tháng | Tỷ lệ DAU/MAU = 40% → user dùng app 12 ngày/tháng |
| **Redemption Rate** | % voucher/phần thưởng được user sử dụng | Phát 100k voucher, 8k được dùng → redemption 8% |
| **Take Rate** | % doanh thu nền tảng giữ lại sau khi trả cho partner/merchant | Shopee take rate ~5-10% trên mỗi giao dịch |
| **MDE (Minimum Detectable Effect)** | Mức thay đổi tối thiểu mà A/B test có thể phát hiện | Nếu MDE = 10%, cần 1 thay đổi >10% mới detect được |
| **Statistical Significance** | % confidence rằng kết quả không phải do ngẫu nhiên | p-value < 0.05 → 95% confidence, kết quả đáng tin |
| **Bounce Rate** | % user vào trang/app rồi rời đi ngay | Landing page bounce 60% → 60% user không làm gì và thoát |
| **Stickiness** | Tỷ lệ DAU/MAU, đo "dính" của user | DAU/MAU = 50% → user dùng app 15 ngày/tháng — rất tốt |
| **Time to Value (TTV)** | Thời gian từ lúc user bắt đầu đến lúc nhận được giá trị đầu tiên | App ngân hàng: từ lúc tải app → nhận được ưu đãi đầu tiên |
| **Net Promoter Score (NPS)** | Đo mức độ sẵn sàng giới thiệu sản phẩm (thang 0-10) | NPS > 50 là xuất sắc; < 0 là báo động |
| **Customer Health Score** | Composite metric tổng hợp đo sức khoẻ của từng user | Dùng frequency + recency + feature usage → 0-100 điểm |

### Tối ưu hoá & Thí nghiệm

| Thuật ngữ | Định nghĩa ngắn | Ví dụ |
|---|---|---|
| **A/B Test (Split Test)** | So sánh 2 biến thể (A và B) trên cùng 1 metric để chọn biến thể tốt hơn | Push A ("Giảm 30%") vs Push B ("Chỉ còn 4 tiếng") |
| **Multivariate Test** | Test nhiều biến cùng lúc (khác A/B test chỉ test 1 biến) | Test cùng lúc: header × CTA × hình ảnh |
| **Conversion Rate (CVR)** | % user hoàn thành hành vi mong muốn | Push → click: 15% → 15% user nhấn vào push |
| **Funnel Analysis** | Phân tích tỷ lệ drop-off qua từng bước | Install (100%) → Register (60%) → First Purchase (20%) |
| **Cohort Analysis** | So sánh hành vi giữa các nhóm user theo thời gian | User tháng 1 có D7 retention 30%, tháng 2 chỉ 25% → có vấn đề |
| **Segmentation** | Chia user thành nhóm dựa trên hành vi/đặc điểm | New user / Active / Dormant / VIP — mỗi nhóm có offer riêng |

### Kinh tế học & Loyalty

| Thuật ngữ | Định nghĩa ngắn | Ví dụ |
|---|---|---|
| **Xu Economy (Point Economy)** | Hệ thống điểm thưởng mà user có thể kiếm và tiêu trong hệ sinh thái | MoMo Xu: kiếm qua thanh toán → tiêu để đổi voucher |
| **Loss Aversion** | User sợ mất hơn là muốn được — tận dụng để giữ chân | "Điểm của bạn sắp hết hạn!" hiệu quả hơn "Nhận thêm điểm!" |
| **Streak (Chuỗi)** | Liên tục hành động trong X ngày liên tiếp — mất chuỗi là mất thành tích | Duolingo: giữ streak 100 ngày → ngại mất nên quay lại mỗi ngày |
| **Gamification** | Dùng cơ chế game (điểm, cấp độ, thử thách) trong ngữ cảnh không phải game | App học: kiếm huy hiệu, lên cấp, challenge bạn bè |
| **VIP Tier (Bậc thành viên)** | Phân hạng user dựa trên chi tiêu/mức độ hoạt động | Silver (0-5tr) → Gold (5-20tr) → Diamond (20tr+) |
| **Lock-in Effect** | User khó rời vì đã đầu tư thời gian/tiền bạc/data vào hệ thống | Đã tích 500k Xu → ngại bỏ app vì mất Xu |
| **Cross-selling (Bán chéo)** | Bán thêm sản phẩm khác cho user hiện tại | User hay order đồ ăn → offer voucher Grab thêm |

### Tăng trưởng & Acquisition

| Thuật ngữ | Định nghĩa ngắn | Ví dụ |
|---|---|---|
| **Viral Coefficient (K-factor)** | Trung bình 1 user giới thiệu được bao nhiêu user mới | K = 1.2 → Cứ 10 user giới thiệu được 12 user mới → tự tăng trưởng |
| **Virality** | Khả năng sản phẩm tự lan truyền qua cơ chế chia sẻ | TikTok: tạo video → share → bạn bè xem → cũng tạo video |
| **Referral Program** | Chương trình thưởng cho user giới thiệu bạn bè | Giới thiệu bạn → cả 2 được 50k |
| **Organic vs Paid** | Organic = user tự tìm đến; Paid = trả tiền để có user | Organic: Google search → Paid: Facebook ads |
| **Retargeting (Remarketing)** | Nhắm lại user đã từng tương tác nhưng chưa chuyển đổi | User click banner nhưng không mua → chạy push nhắc lại sau 3 ngày |
| **Lookalike Audience** | Facebook/X tự tìm user giống với những user tốt nhất của bạn | Upload 10k VIP user → Facebook tìm thêm 100k user tương tự |
| **Zero-Party Data** | User chủ động cung cấp dữ liệu (sở thích, nhu cầu) | Khi đăng ký: "Bạn thích nhận ưu đãi gì?" → user tự chọn |
| **First-Party Data** | Dữ liệu thu thập từ tương tác trực tiếp của user với sản phẩm | Lịch sử giao dịch, hành vi trong app, push response |
| **Super-App Strategy** | Biến app chuyên dụng thành nền tảng đa dịch vụ | Grab: book xe → gọi đồ ăn → thanh toán → bảo hiểm |

### Kỹ thuật & Automation

| Thuật ngữ | Định nghĩa ngắn | Ví dụ |
|---|---|---|
| **RAG (Retrieval-Augmented Gen)** | AI tự tra cứu kiến thức (từ database) trước khi trả lời, thay vì chỉ dựa vào kiến thức có sẵn | Campaign search engine: user query → TF-IDF search campaigns → AI rút ra đề xuất |
| **Embedding** | Biến văn bản thành vector số — văn bản "gần nghĩa" có vector gần nhau | "Giảm giá" và "khuyến mãi" có embedding gần nhau, dù từ khác nhau |
| **TF-IDF** | Kỹ thuật đo độ quan trọng của từ trong document — từ hiếm = quan trọng hơn | "Streak" trong campaign pool chỉ 1 campaign có → từ này rất đặc trưng cho campaign đó |
| **n8n / Make / Zapier** | No-code automation platform — nối các service với nhau không cần code | n8n: Google Sheet → OpenAI → Telegram → auto-report |
| **Webhook** | Tự động gửi dữ liệu khi có sự kiện — thay vì phải hỏi liên tục | User dùng voucher → webhook gửi data vào Google Sheet |
| **API (Application Programming Interface)** | Cổng kết nối giữa các hệ thống — cho phép 2 app nói chuyện với nhau | Google Sheets API: đọc/ghi sheet từ code hoặc n8n |
| **Agent (AI Agent)** | AI tự động thực hiện task — không chỉ trả lời, mà còn đọc data, quyết định, hành động | Growth Monitor Agent: check metric → phân tích → gửi alert → tự động |

## Prompt duy nhất — Giải thích thuật ngữ bằng ví dụ campaign của bạn

**Khi nào dùng:** Gặp thuật ngữ growth lạ, muốn hiểu ngay bằng ví dụ thực tế của mình. Hoặc cần giải thích cho team/sếp không có background growth.

```
Giải thích thuật ngữ growth sau đây:

Thuật ngữ: [tên, VD: North Star Metric / Growth Loop / JTBD / LTV / CAC / MDE]

Yêu cầu:
1. Định nghĩa ngắn — 1 câu, không jargon
2. Ví dụ từ công ty thật (Duolingo, Shopee, Grab, Spotify...)
3. Ví dụ áp dụng vào campaign của tôi: [mô tả ngắn campaign/sản phẩm]
4. Khi nào KHÔNG nên dùng thuật ngữ/framework này
5. Nếu cần giải thích cho sếp không có background growth: dùng analogy gì?
```

**Biến thể — So sánh 2 thuật ngữ dễ nhầm:**

```
Phân biệt 2 thuật ngữ:

Thuật ngữ A: [tên A, VD: Retention]
Thuật ngữ B: [tên B, VD: Engagement]

1. Định nghĩa từng cái (1 câu)
2. Sự khác biệt chính — bảng so sánh
3. Khi nào đo A, khi nào đo B
4. Ví dụ ngắn minh họa sự khác biệt
```

**Biến thể — Chuẩn bị từ vựng cho presentation:**

```
Tôi sắp thuyết trình về [chủ đề] cho [audience: team tech / C-level / new hire].

Top 5 thuật ngữ growth nên dùng trong buổi này:
1. Định nghĩa 1 câu cho từng thuật ngữ (nói được trong meeting)
2. 1 câu ví dụ gắn với campaign của tôi
3. Thuật ngữ nào KHÔNG nên dùng với audience này (quá technical / dễ hiểu sai)
```

## 📖 Glossary

### Strategy & Frameworks

| Term | Definition | Example |
|---|---|---|
| **AARRR (Pirate Metrics)** | 5-stage user journey: Acquisition → Activation → Retention → Revenue → Referral | Diagnose which stage is your bottleneck |
| **North Star Metric** | One metric that captures the core value users get from your product | Duolingo: "Minutes learned per day" — not "Signups" |
| **Growth Loop** | Self-reinforcing system where output of one cycle feeds the next | Dropbox: Use → Share → Get space → Use more |
| **JTBD (Jobs to Be Done)** | Users "hire" products to do a specific job in their lives | People hire Grab not to "ride a car" but to "get from A to B without worrying" |
| **ICE/RICE** | Experiment prioritization: Impact × Confidence × Ease (ICE) or Reach × I × C × Effort (RICE) | Campaign with I=8, C=6, E=7 → ICE = 336 |
| **Growth Funnel** | User steps from awareness to conversion | Install → Register → First Action → Retention → Purchase |
| **Flywheel** | Customer satisfaction → referral → new customers → more satisfaction | HubSpot: content marketing → read → trust → buy → recommend |
| **TAM/SAM/SOM** | TAM = Total Addressable Market, SAM = Serviceable Available, SOM = Serviceable Obtainable | Estimate campaign scale and justify budget |

### Metrics & Measurement

| Term | Definition | Example |
|---|---|---|
| **CAC** | Total marketing cost / New users acquired | Campaign costs 100M VND, gets 10k users → CAC = 10k |
| **LTV** | Average revenue a user generates over their lifetime | Game user: top-up 50k/month × 12 months = LTV = 600k VND |
| **Retention Rate** | % of users who return after X days | D7 retention = 30% → 30% of users return within 7 days |
| **Churn Rate** | % of users who stop using the product in a given period | 5% monthly churn → ~46% annual churn |
| **DAU/MAU** | Daily / Monthly Active Users | DAU/MAU = 40% → user opens app 12 days/month |
| **Redemption Rate** | % of vouchers/rewards actually used by users | 100k vouchers issued, 8k used → 8% redemption |
| **MDE** | Minimum Detectable Effect — smallest change an A/B test can detect | MDE = 10% means you can only detect changes >10% |
| **Statistical Significance** | % confidence that results aren't due to random chance | p-value < 0.05 → 95% confident, result is real |
| **Stickiness** | DAU/MAU ratio — measures "stickiness" | DAU/MAU = 50% → user opens 15 days/month |
| **Time to Value** | Time from user start to receiving first value | Banking app: from download → receiving first offer |
| **Customer Health Score** | Composite metric measuring individual user health | Frequency + recency + feature usage → score 0-100 |

### Optimization & Experimentation

| Term | Definition | Example |
|---|---|---|
| **A/B Test** | Compare 2 variants (A vs B) on the same metric | Push A ("30% off") vs Push B ("Only 4 hours left") |
| **Conversion Rate** | % of users completing target action | Push → click: 15% |
| **Funnel Analysis** | Analyze drop-off rates across user journey steps | Install (100%) → Register (60%) → First Purchase (20%) |
| **Cohort Analysis** | Compare behavior across user groups over time | January users D30 retention = 30%, February = 25% → problem detected |
| **Segmentation** | Group users by behavior/characteristics | New / Active / Dormant / VIP — different offers per group |

### Economics & Loyalty

| Term | Definition | Example |
|---|---|---|
| **Point Economy** | Reward system where users earn and spend points within ecosystem | MoMo Xu: earn via payments → spend on voucher redemption |
| **Loss Aversion** | Users fear losing more than they desire gaining | "Your points are expiring!" works better than "Earn more points!" |
| **Streak** | Consecutive days of action — breaking the streak loses progress | Duolingo: 100-day streak → don't want to lose it |
| **Gamification** | Applying game mechanics (points, levels, challenges) in non-game contexts | Learning app: earn badges, level up, challenge friends |
| **VIP Tier** | User tiering based on spend/activity | Silver (0-5M) → Gold (5-20M) → Diamond (20M+) |
| **Lock-in Effect** | Hard for users to leave because of invested time/money/data | Accumulated 500k Xu → reluctant to leave app |
| **Cross-selling** | Selling additional products to existing users | Food delivery user → offer Grab ride voucher |

### Growth & Acquisition

| Term | Definition | Example |
|---|---|---|
| **K-factor** | Avg number of new users each existing user brings in | K=1.2 → every 10 users bring 12 new users → organic growth |
| **Virality** | Product's ability to spread through sharing mechanisms | TikTok: create → share → friends watch → also create |
| **Referral Program** | Rewarding users for referring friends | Refer a friend → both get 50k VND |
| **Retargeting** | Re-engaging users who interacted but didn't convert | User clicked banner but didn't buy → push reminder after 3 days |
| **Zero-Party Data** | Data users proactively share (preferences, needs) | During signup: "What offers interest you?" → user chooses |
| **First-Party Data** | Data collected from direct user-product interaction | Transaction history, in-app behavior, push response |
| **Super-App Strategy** | Turning a specialized app into a multi-service platform | Grab: ride-hailing → food → payments → insurance |

### Technical & Automation

| Term | Definition | Example |
|---|---|---|
| **RAG** | AI retrieves knowledge from database before answering | Campaign search engine: query → search campaigns → AI recommends |
| **Embedding** | Convert text to numeric vectors — semantically similar text has close vectors | "Discount" and "promotion" have close embeddings |
| **TF-IDF** | Measures word importance in a document — rare words carry more weight | "Streak" appears in only 1 campaign → highly distinctive |
| **n8n / Make / Zapier** | No-code automation platforms — connect services without code | n8n: Google Sheet → OpenAI → Telegram → auto-report |
| **Webhook** | Auto-send data when an event occurs | User redeems voucher → webhook sends data to Google Sheet |
| **API** | Connection point between systems — allows apps to talk to each other | Google Sheets API: read/write from code or n8n |
| **AI Agent** | AI that autonomously performs tasks — reads data, decides, acts | Growth Monitor Agent: check metric → analyze → alert → act |

## Single prompt — Explain a term using your campaign context

**When to use:** Hit an unfamiliar growth term and want a practical explanation. Or need to explain it to a non-growth stakeholder.

```
Explain this growth term:

Term: [name, e.g., North Star Metric / Growth Loop / JTBD / LTV / CAC / MDE]

Requirements:
1. One-line definition — no jargon
2. Real company example (Duolingo, Shopify, Uber, Spotify...)
3. Apply to my context: [brief description of your campaign/product]
4. When NOT to use this term or framework
5. If explaining to a non-growth executive: what analogy works best?
```

**Variant — Compare 2 confusing terms:**

```
Distinguish between:

Term A: [e.g., Retention]
Term B: [e.g., Engagement]

1. One-sentence definition each
2. Key differences — comparison table
3. When to measure A vs B
4. Short example illustrating the difference
```

**Variant — Prep vocabulary for a presentation:**

```
I'm presenting about [topic] to [audience: tech team / C-level / new hires].

Top 5 growth terms to use:
1. One-sentence definition per term (speakable in a meeting)
2. One example sentence linked to my campaign
3. Terms to AVOID with this audience (too technical / easily misunderstood)
```
